# Growth Infrastructure

## Goal
Set up the `growth/` directory as the operational hub for all marketing and content work. Templates, drafts, scripts, checklists, and video outlines — everything organized so content creation is fast and repeatable.

## Why Now
Without structure, content creation becomes ad-hoc and drops off after week 1. The infrastructure ensures consistent output for 12 weeks.

## Status
Done as of 2026-03-28. Project status is satisfied by completed task `6.1`; task `6.2` was cancelled after its deliverable was already represented inside the `growth/` workspace and no longer needed separate tracking.

## In Scope
- `growth/` directory structure at repo root
- Post templates and drafts (X threads, Reddit, Discord)
- Article drafts and outlines
- Video scripts and recording checklists
- Submission trackers (which platforms, when, status)
- Weekly metrics tracking template
- Content calendar

## Out Of Scope
- Automation tools or scripts for posting
- Analytics dashboards
- CRM or user tracking

## Shared Context
The `growth/` directory is gitignored or kept on a separate branch if preferred. It's an operational tool, not part of the distributed skill project.

## Dependency Notes
Should be set up before or in parallel with project 4 (content creation).

## Success Definition
Any content task from project 4 can be started by opening the relevant template in `growth/` and filling it in. Weekly metrics review takes < 10 minutes.

- Create submission tracker
