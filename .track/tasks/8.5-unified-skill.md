## Context
Consolidate 9 skills into 1 unified `track` skill (auto-discovered in `.track/` repos). This skill handles all human-intent interactions:

| User intent | Mode |
|-------------|------|
| "What should I work on?" | pick — read state, pick task |
| "Start task 4.1" | start — create branch, open draft PR, emit event |
| "Create a task for X" | create — create task file, validate, regenerate |
| "Break this into tasks" | decompose — explore codebase, propose breakdown |
| "Link this to task 4.1" | link — emit track.link event for retroactive attribution |
| "Add context: found that..." | context — append to task Notes |
| "What's the status?" | status — query server API or regenerate views |

The skill detects mode from user intent (natural language parsing, not explicit mode flags).

Content merges from:
- `skills/work/SKILL.md` → pick, start, context modes
- `skills/create/SKILL.md` → create mode
- `skills/decompose/SKILL.md` → decompose mode

New modes: link, status.

Hook-aware lifecycle: when git hooks are detected, skip manual status writes. When absent, fall back to current PR-based lifecycle.

## Acceptance Criteria
- [ ] `skills/track/SKILL.md` exists with all 7 modes
- [ ] Auto-discovers in repos with `.track/` directory
- [ ] Each mode has Definition of Done and Closing Message
- [ ] Hook-aware: detects git hooks, adjusts lifecycle behavior
- [ ] "Do Not" section covers mode drift, scope violations
- [ ] Existing work/create/decompose test cases still conceptually covered
