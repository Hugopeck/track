## Context
Promptfoo has an `anthropic:claude-agent-sdk` provider for evaluating Claude Code agent workflows. It supports tool-call assertions, LLM-as-judge grading, session resume, and cost capping. This could replace or complement the bash + `claude -p` approach for CI-ready skill testing.

Questions to answer:
- Can the provider load Track as a plugin via `plugins` config?
- What assertions work for skill protocol compliance (tool-call ordering, mode lock)?
- Is the cost/run acceptable for CI? (estimate tokens per skill test)
- How does it handle non-determinism across runs?
- What's the setup cost (install, config, learning curve)?
