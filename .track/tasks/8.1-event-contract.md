## Context
The event contract is the key interface between this repo (protocol, skills, hooks) and the track-server repo (Bun/TS binary). It defines event types, JSON payload schemas, the attribution model, and the LLM interface spec.

This must be written first because hook templates (8.3) and the server (8.7) both implement against this contract.

Key design decisions already resolved:
- LLM-vendor-agnostic: OpenAI-compatible `/v1/chat/completions`
- Config: `base_url` + `api_key` + `model` (no vendor lock-in)
- Attribution is nullable: `null` = untracked activity, not lost
- Event types: track.commit, track.pr.opened, track.pr.ready, track.pr.merged, track.task.started, track.link

Reference: `.track/plans/track-oss-alignment.md` section 1B.
