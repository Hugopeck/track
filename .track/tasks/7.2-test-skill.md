## Context
Track needs an internal skill that acts as an intelligent test orchestrator. It runs both bash script tests and agent-driven skill tests, interprets failures semantically, and produces structured reports. This is the centerpiece of the testing project.

Operating modes:
- `scripts` — run bash test suite, interpret failures, suggest fixes
- `skills` — run skill smoke tests via `claude -p` in isolated worktrees, validate tool-call chains
- `full` — run both, produce unified report
- `single` — test a specific named skill or script

Key design:
- Each skill test runs in an isolated worktree (no side effects on real repo)
- Skill tests validate: correct mode lock, tool-call sequence, closing message format, "Do Not" compliance
- Budget cap per skill invocation (`--max-budget-usd 0.50`)
- Non-determinism handled by testing invariants (e.g., "validate was called") not exact output
- Uses `claude -p --plugin-dir . --output-format json --bare` for headless skill invocation

## Acceptance Criteria
- [ ] `skills/test/SKILL.md` follows the standard skill protocol structure (modes, definition of done, closing message matrix, do not)
- [ ] `scripts` mode runs `tests/run-all.sh`, parses output, classifies failures
- [ ] `skills` mode invokes at least `/track:validate` and `/track:todo` headlessly and checks protocol compliance
- [ ] `full` mode runs both and produces a unified pass/fail report
- [ ] `single` mode accepts a skill or test name argument
- [ ] Allowed tools include Bash, Read, Glob, Grep, Agent
