## Context
A 5-10 minute video showing Track in action with multiple agents in Conductor. This is the primary deep-dive asset for people who want to understand Track before installing.

## Video outline
**Title:** "Linear just proved project management is dead. Here's what replaces it."

1. **Hook (30s):** "Linear just admitted issue tracking was built for handoffs. But their answer is still a SaaS tool. Mine is a folder."
2. **The problem (1m):** Show what happens when 3 agents work without coordination — file conflicts, duplicate PRs, wasted compute
3. **The solution (30s):** Explain Track in one sentence. Show the `.track/` folder.
4. **Demo (3m):**
   - `/track:setup-track` in a real repo
   - `/track:decompose` a real task into subtasks
   - Open Conductor, launch 3 agents on different tasks
   - Show agents working in parallel, each on their own files
   - PRs auto-created, TODO.md updating
   - Merge a PR → task auto-completes
5. **The vision (30s):** "If a folder can manage a codebase, why not anything? Wedding, book, renovation. A folder is all you need."
6. **CTA (15s):** Star on GitHub, one-line install, follow on X

## Recording tips
- Screen record with OBS or QuickTime
- Voiceover can be added later or done live
- Show real terminal, real Conductor, real GitHub
- Edit to keep it tight — cut dead air and loading screens
