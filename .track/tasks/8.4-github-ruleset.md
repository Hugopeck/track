## Context
Create a GitHub Ruleset JSON template that init deploys to enforce Track's CI checks as merge gates on main/master.

The ruleset enforces:
- No branch deletion, no force push, linear history required
- PR required (0 approvals for solo devs, dismiss stale reviews)
- Required status checks: `track-validate`, `track-pr-lint`

Deployed via `gh api -X POST /repos/:owner/:repo/rulesets --input track-ruleset.json`. Gated on repo admin access — if not available, skip with clear message.

This is the "rulesets over branch protection" approach: version-controllable, scalable, supports evaluate mode.
