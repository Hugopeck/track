## HN post draft

**Title:** "Show HN: Track — git-native task coordination for AI coding agents"

**Text:**
Track is a task coordination system for AI coding agents (Claude Code, Cursor, Codex, etc.). It's just a `.track/` folder in your git repo — markdown task files with YAML frontmatter, bash scripts for validation, and PR-driven status updates.

The problem: when you run multiple AI agents on the same codebase, they edit the same files, create duplicate PRs, and waste compute. Track solves this by giving each task a `files:` field that declares which files it touches, so agents know what's available.

It's ~600 lines of bash, zero dependencies beyond git, and works with any agent that can read files. No server, no accounts.

GitHub: [link]

I built this because I was running 3-5 Claude Code agents in parallel and kept hitting coordination failures. Linear just launched their own AI agent yesterday, which validated the problem — but their answer is still "use our SaaS tool." I think the filesystem is the tool.

Happy to answer questions about the approach, the protocol design, or multi-agent coordination in general.
