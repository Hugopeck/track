## Context
Skills are LLM protocols with zero automated testing today. The `claude -p` flag enables headless, scriptable invocation with `--output-format json` for machine-parseable results. This task proves the pattern by smoke-testing each skill.

Invocation pattern:
```bash
claude -p "prompt" --plugin-dir . --output-format json --bare \
  --allowedTools "Bash,Read,Glob,Grep" --max-budget-usd 0.50
```

The JSON output includes tool calls and result text, which can be parsed with `jq` to validate protocol compliance.

## Acceptance Criteria
- [ ] `/track:validate` on a valid fixture → exits cleanly, output shows `track-validate.sh` was called
- [ ] `/track:todo` on a valid fixture → generates TODO.md, output confirms generation
- [ ] `/track:create` with a structured prompt → creates a task file in `.track/tasks/`
- [ ] `/track:work` with no available tasks → reports nothing to do (does not hallucinate work)
- [ ] Each test validates tool-call presence in the JSON output (not exact text)
- [ ] Tests run in isolated temp directories to avoid polluting the real repo
- [ ] Budget cap per invocation prevents runaway costs
