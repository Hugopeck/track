## Context
Signal the future paid direction early in the README. This reassures users the free tier isn't going away, attracts enterprise-curious users, and lets us gauge interest through GitHub issues and discussions.

## What to add
A "Roadmap" section in README.md with three tiers:

1. **Free forever (core):** Single-repo coordination, all agent platforms, bash scripts, markdown protocol
2. **Coming — Track Pro:** Cross-repo dashboards, team analytics, Slack notifications, audit trails
3. **Coming — Track for Teams:** Approval workflows, permissions, compliance reporting

Keep it concise — a small table or 3-bullet list. Don't oversell vaporware. Frame as "where we're headed" not "coming soon."
