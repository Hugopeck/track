## Context
GitHub Discussions is the lightweight community hub — no Discord server to maintain, integrated with the repo. Needs to be live before launch so visitors have a place to engage.

## Steps
1. Enable Discussions in repo settings (Settings → Features → Discussions)
2. Create categories:
   - **General** — introductions, questions, chat
   - **Show & Tell** — "I'm using Track for X" posts
   - **Ideas** — feature requests and suggestions
   - **Q&A** — technical questions (mark as answered)
   - **Track for Non-Code** — the "everything is PM" use cases
3. Pin a welcome post explaining what Track is and linking to install
4. Pin a "What are you managing with Track?" prompt

## Acceptance Criteria
- [ ] GitHub Discussions enabled
- [ ] Categories created
- [ ] Welcome post pinned
- [ ] "What are you managing?" prompt pinned
