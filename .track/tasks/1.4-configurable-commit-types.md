## Context

The conventional-commit-lint script hardcodes 7 types. Two problems:
1. Missing standard types: `perf`, `style`, `build`, `revert`
2. Adopting repos with custom types (e.g., `wip`, `security`) have no override path

Fix: expand the default list to cover the full Conventional Commits spec, and add
optional config via `.track/config.yml` so adopting repos can override the type list.

## Acceptance Criteria
- [x] Default type list expanded to include `perf`, `style`, `build`, `revert`
- [x] Script reads `commit_types:` list from `.track/config.yml` when present
- [x] Config file takes precedence over defaults
- [x] If config is present but `commit_types` is missing/empty, falls back to defaults
- [x] bash 3.2+ compatible
- [x] `.track/scripts/track-conventional-commit-lint.sh` (deployed copy) updated to match
