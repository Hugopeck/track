## Context
Each Good First Issue is a potential contributor who stars + shares. Create approachable issues that invite the community to expand Track's scope.

## Issue ideas

**Easy (docs/templates):**
1. "Add Track getting-started guide for Windsurf" — research + write docs
2. "Add Track getting-started guide for Aider" — research + write docs
3. "Create example .track/ for a non-code project" — creative writing
4. "Improve error messages in track-validate.sh" — bash, small scope
5. "Add --json output flag to track-todo.sh" — bash, well-scoped

**Medium (features):**
6. "Track for monorepos — support nested .track/ directories"
7. "Add task templates for common project types"
8. "Support task labels/tags in frontmatter"
9. "Add task time tracking (started/completed timestamps)"
10. "Color-coded TODO.md output for terminal"

**Community/creative:**
11. "Track for [your thing] — share your non-code .track/ setup"
12. "Design a Track logo/icon"

## Acceptance Criteria
- [ ] 10+ issues created on GitHub with "good first issue" label
- [ ] Each issue has clear description, acceptance criteria, and helpful context
- [ ] Mix of difficulty levels (easy/medium)
- [ ] At least 2 "Track for X" community issues

Created from launch plan Phase 3. Use `gh issue create` for each.
