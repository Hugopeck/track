## Context
Two git hook templates that init deploys to adopting repos:

**commit-msg** — bash-only conventional commit linter.
- Regex: `^(feat|fix|docs|refactor|test|ci|chore)(\(.+\))?!?: .+`
- Pure bash 3.2+, zero external deps
- Clear error message showing expected format on rejection

**post-commit** — event emitter.
- Extracts changed files via `git diff --name-only HEAD~1`
- Parses conventional commit type/scope/subject from commit message
- Calls `track_match_files_to_task()` from track-common.sh (8.2)
- POSTs JSON event to `localhost:${TRACK_PORT:-4747}/events` if server responds
- Falls back to appending JSON line to `.track/events/log.jsonl` (gitignored)
- Never blocks the commit — fire-and-forget
- Event payload follows `.track/specs/event-contract.md` (8.1)

Both hooks live in `skills/init/assets/hooks/`.
