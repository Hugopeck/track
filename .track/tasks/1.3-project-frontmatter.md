## Context
Project briefs currently have no frontmatter — they're pure markdown with required sections. Tasks have structured YAML frontmatter but projects don't, which means there's no machine-readable way to sort, filter, or prioritize projects.

Add YAML frontmatter to project briefs with at least:
```yaml
---
id: "8"
title: "Track-OSS Alignment"
priority: high
status: active
created: 2026-03-30
updated: 2026-03-30
---
```

Fields:
- `id`: project number (derived from filename, now explicit)
- `title`: project name (matches H1)
- `priority`: `urgent | high | medium | low`
- `status`: `planning | active | done | paused`
- `created` / `updated`: dates

Update:
- `projects-readme.md` template to document the frontmatter format
- `track-validate.sh` to validate project frontmatter (if it doesn't already)
- `track-common.sh` if it needs a project frontmatter parser
- All existing project briefs in `.track/projects/` to add frontmatter
