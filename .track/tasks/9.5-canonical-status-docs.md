## Context
Project 9 changes the mental model as much as the runtime. Track docs currently teach a split raw/effective status story and manual agent-owned transitions for `active` and `review`.

This task updates the protocol and repo guidance so every supported surface describes one canonical task status, script-owned lifecycle movement, validation-after-sync, and reconciliation as the recovery path. The documentation should explain the new model without reintroducing a second status system in user-facing language.

- [ ] The docs describe lifecycle sync, validation ordering, and reconciliation as the supported operating model
- [ ] Documentation changes stay consistent with the implemented runtime and workflow behavior from the preceding project 9 tasks
