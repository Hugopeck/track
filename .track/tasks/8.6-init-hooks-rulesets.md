## Context
Extend the init skill with two new deployment phases:

**Hook deployment:**
- Deploy `commit-msg` and `post-commit` from `skills/init/assets/hooks/` to `.git/hooks/`
- If `.husky/` exists, deploy there instead
- Make executable
- Add `.track/events/` directory (gitignored) for event log

**Ruleset deployment:**
- Apply `track-ruleset.json` via `gh api -X POST /repos/:owner/:repo/rulesets`
- Gate on repo admin access — detect with `gh api /repos/:owner/:repo` and check permissions
- If not admin, skip with clear message explaining what the ruleset would do

Update `install-manifest.json` with new `hooks` section.

## Acceptance Criteria
- [ ] Init skill deploys commit-msg and post-commit hooks
- [ ] Hooks deployed to `.git/hooks/` or `.husky/` as appropriate
- [ ] Init skill applies GitHub Ruleset when admin access available
- [ ] Graceful skip with message when not admin
- [ ] `install-manifest.json` updated with hooks section
- [ ] `.track/events/` created and gitignored
