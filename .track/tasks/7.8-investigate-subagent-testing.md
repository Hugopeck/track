## Context
Claude Code sub-agents with `isolation: worktree` provide clean filesystem isolation for testing skills that modify repo state. The Agent tool can spawn sub-agents with restricted tools, custom prompts, and isolated context. This could enable the `/track:test` skill to test other skills in parallel worktrees without `claude -p`.

Questions to answer:
- Can a parent agent spawn sub-agents that load skills and run them?
- How does `isolation: worktree` interact with `--plugin-dir` / plugin loading?
- Can the `/track:test` skill use the Agent tool to test other skills in parallel worktrees?
- What's the token/cost overhead of sub-agent testing vs `claude -p`?
- Can sub-agents report structured results back to the parent?
- What are the limitations? (sub-agents cannot spawn other sub-agents)
