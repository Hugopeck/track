## Context
The first 50 active users become "Track Pioneers" — an informal early adopter group that influences the roadmap and evangelizes Track. This creates belonging and retention.

## Design decisions
1. **Channel:** GitHub Discussions (lowest friction) or private Discord channel?
2. **Entry criteria:** Anyone who has `.track/` in a real repo? First 50 to star + use?
3. **Benefits:**
   - "Pioneer" label on GitHub Discussions
   - Direct input on roadmap
   - Early access to Track Pro features (when they exist)
   - Mentioned in README contributors section
4. **Expectations:** Share what you're using Track for, report bugs, suggest features

## Acceptance Criteria
- [ ] Program designed and documented
- [ ] Entry criteria defined
- [ ] Announcement post drafted
- [ ] GitHub Discussion or channel created for Pioneers

## Notes
Keep it lightweight. The goal is community, not overhead.
