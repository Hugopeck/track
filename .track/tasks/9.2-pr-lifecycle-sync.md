## Context
Once shared status-write helpers exist, Track needs a single PR lifecycle sync path that translates GitHub pull request events into canonical task status changes.

This task owns the runtime and deployment layer: the sync script, local wrapper commands, and the workflow/setup-track asset changes needed to install and run the new lifecycle flow in both this repo and adopting repos. It should preserve same-repo write behavior and fallback semantics while ensuring status sync runs before downstream checks.

## Acceptance Criteria
- [ ] `track-sync-pr-status.sh` maps PR lifecycle events to canonical task status transitions for draft open, ready-for-review, converted-to-draft, reopen, and closed-unmerged cases
- [ ] Local wrapper commands exist for starting work and marking ready for review through the lifecycle sync path
- [ ] Workflow definitions in both `.github/workflows/` and `skills/setup-track/assets/workflows/` install the ordered PR lifecycle sync path
- [ ] `skills/setup-track/assets/install-manifest.json` deploys any new runtime scripts required by the lifecycle sync flow
