## Context
When an agent pushes with invalid Track state, it fails in CI. A pre-push hook that runs `track-validate.sh` locally would catch errors before they hit GitHub — faster feedback, fewer wasted CI runs.

## Acceptance Criteria
- [ ] `pre-push` hook runs `track-validate.sh` and blocks push on failure
- [ ] Hook deployed via `setup-track` (install-manifest + assets)
- [ ] Existing tests pass, new hook tested
