## Context
Release Please PRs are created and updated by GitHub Actions, but the required Track checks are only guaranteed to run on normal pull request events. PR #64 sat blocked with no check runs because the required contexts never attached to the release branch head SHA.

This task fixes both sides of the gap: normal PRs should run Track checks on open/reopen, and Release Please PRs should have the required contexts written even when GitHub suppresses follow-on workflow runs from bot-created PR events.

## Acceptance Criteria
- [ ] `Track Validate` and `Track PR Lint` run on PR open/reopen events for normal pull requests
- [ ] Release Please writes `Track Validate`, `Track PR Lint`, and `conventional-commit-lint` statuses to the release PR head SHA
- [ ] Release PR checks execute against the release PR branch contents with the same PR metadata/env shape as the regular workflows
- [ ] `bash .track/scripts/track-validate.sh` passes after the task file and workflow changes
