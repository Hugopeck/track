## Context
No existing test exercises the full create→validate→todo→complete chain. Each script is tested in isolation with its own fixtures. An E2E test scaffolds a complete `.track/` from scratch, writes task files, runs validate, generates TODO.md, then runs complete — verifying the handoffs between scripts work correctly.

## Acceptance Criteria
- [ ] Test creates a temp repo with `.track/` structure (projects + tasks in multiple statuses)
- [ ] Runs `track-validate.sh` and asserts exit 0
- [ ] Runs `track-todo.sh --local --offline` and verifies TODO.md contains correct project grouping and task sorting
- [ ] Simulates completion via `track-complete.sh` with a task branch name and PR URL
- [ ] Verifies the completed task file has `status: done`, correct PR URL, and updated date
- [ ] Re-runs `track-validate.sh` after completion and asserts exit 0
- [ ] Tests at least one dependency chain: task B depends on A, A gets completed, then B appears in immediate starts
