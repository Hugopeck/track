## Context
Document the track-server architecture in enough detail to start building it in a separate repo. This is a planning task — produce a specification, not code.

**Server stack:** Bun/TypeScript, `bun compile` for binary, `bun:sqlite` built-in.

**Key components:**
- HTTP server (event ingestion, REST API, dashboard serving)
- SQLite schema (tasks, events with nullable attribution, activity_summary)
- Matcher pipeline (deterministic first, LLM fallback for ambiguous)
- LLM integration (raw fetch against OpenAI-compatible API, no SDK)
- Task state sync (reads `.track/tasks/*.md` as source of truth)
- Dashboard (single-page HTML on same port)
- Install script (launchd/systemd registration)

**Architecture details in plan:** `.track/plans/track-oss-alignment.md` Part 2 has the full spec including directory layout, SQLite schema, processing pipeline, TypeScript interfaces, and install flow.

This task produces a standalone `SERVER-ARCHITECTURE.md` at repo root that the track-server repo can use as its founding spec.
