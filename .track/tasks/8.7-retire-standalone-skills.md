## Context
Task 8.7 originally assumed a unified `track` dispatcher from 8.5. That architecture was dropped. The revised scope is: retire script-only skills that provide no user value as skills, rename utility skills for clarity, and document the no-dispatcher model.

**Remove SKILL.md from** (scripts stay, skill protocol goes):
- `skills/validate/` — script-only (runs in hooks + CI)
- `skills/test/` — internal dev tool, not shipped

**Keep as standalone skills:**
- `skills/work/` — core workflow, still auto-loaded in Track repos
- `skills/create/` — separate task/project creation workflow
- `skills/decompose/` — separate planning/decomposition workflow
- `skills/todo/` — renamed skill `refresh-track`
- `skills/update-track/` — renamed skill `update-skills`, auto-runs at session start

**Keep as non-skills:**
- `skills/runtime/` — shared bash library (not a skill)
- All `scripts/` subdirectories — enforcement scripts stay
- `skills/init/SKILL.md` — the setup skill

Update skill docs and tests to reflect the revised retained-skill set.
