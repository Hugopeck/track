## Context
Even with ordered PR lifecycle sync, Track still needs a recovery path for missed events and a clear way to communicate when GitHub-derived checks are unavailable.

This task owns the read/repair layer. It should add a reconciler that repairs safe canonical-status mismatches from live PR state, while updating `track-todo.sh` to render canonical task status and warn clearly when GitHub lookup is offline, partial, or stale rather than silently substituting a second user-facing status model.

## Acceptance Criteria
- [ ] `track-reconcile.sh` scans tasks and open PRs, repairs safe canonical-status mismatches, and reports unresolved conflicts for manual follow-up
- [ ] `track-todo.sh` renders canonical task status without presenting raw/effective status as separate user-facing concepts
- [ ] Generated views warn clearly when GitHub PR lookup is unavailable, partial, or stale
- [ ] `tests/test-todo.sh` and `tests/test-todo-extended.sh` cover stale-warning behavior and safe reconciliation cases
