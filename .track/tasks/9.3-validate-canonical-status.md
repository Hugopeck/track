## Context
Validation currently encodes assumptions from the manual-status model, which creates false failures when PR state changes and task-file writes are not already in sync.

This task updates validation to enforce invariants in the new ordered lifecycle model: canonical status should be checked after sync, not as a manual prerequisite. Duplicate PR detection, orphaned PR handling, blocked/terminal rules, and dependency checks must remain intact while race-sensitive checks are redesigned.

## Acceptance Criteria
- [ ] `track-validate.sh` no longer assumes agents manually edited `active` or `review` before PR-triggered automation runs
- [ ] Validation still rejects duplicate PR mappings, orphaned PRs, blocked tasks with open PRs, and terminal tasks with open PRs
- [ ] PR-lifecycle validation aligns with the ordered sync model instead of failing on pre-sync timing windows
- [ ] Validation coverage in `tests/test-validate.sh`, `tests/test-validate-extended.sh`, and `tests/test-e2e-lifecycle.sh` reflects the canonical-status model and passes
