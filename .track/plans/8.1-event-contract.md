# Task 8.1 Event Contract Plan

## Summary

Write the event contract as a single-repo internal spec for this repository only at `.track/specs/event-contract.md`.

This spec defines the contract between Track's local emitters in this repo (`hooks`, `skills`, scripts) and Track's future local runtime/server described by project 8.

## Scope

- Create `.track/specs/event-contract.md`
- Define all 6 event types with explicit JSON schemas
- Define the shared common envelope
- Define the server-enriched attribution/result model separately from emitted wire events
- Define the OpenAI-compatible LLM attribution interface
- Define versioning and compatibility rules for this repo's future runtime

## Structure

1. Title and purpose
2. Scope / non-goals
3. Contract role inside this repo
4. Transport contract
5. Common event envelope
6. Event type summary table
7. Six event sections with example + schema
8. Attribution/result model
9. LLM interface
10. Versioning and compatibility
11. References to downstream tasks (`8.3`, `8.5`, `8.9`)

## Key decisions

- One repository only: this repo
- Canonical spec path: `.track/specs/event-contract.md`
- Contract shape: two-layer model
  - wire events
  - runtime-enriched attribution result
- PR events carry `primary_task_id`
- Commit events do not require final attribution fields on ingress
- `track.link` v1 supports `branch_history` linkage only
- Transport response body is intentionally unspecified in v1
- Executable contract tests belong to task `8.3`
