# Task 8.2 — Deterministic scope matching in `track-common.sh`

## Summary
Implement a bash-3.2-compatible helper in `skills/runtime/scripts/track-common.sh` that matches changed file paths to Track task `files:` globs using the existing prefix-based glob overlap logic. The matcher will consider only open tasks (`todo`, `active`, `review`), ignore terminal tasks (`done`, `cancelled`), and expose results via globals plus exit status so the future `post-commit` hook can consume it without parsing stdout.

## Public API / Interface Additions
Add these to `skills/runtime/scripts/track-common.sh`:

### New globals
- `TRACK_MATCHED_TASK_IDS=()` — array of matched task IDs in deterministic order.
- `TRACK_MATCHED_TASK_IDS_SERIALIZED=''` — serialized form of the matched IDs via `track_serialize_items`.
- `TRACK_MATCH_CONFIDENCE=''` — one of `deterministic`, `ambiguous`, `unmatched`.
- `TRACK_MATCH_ERROR=''` — non-empty only when the matcher cannot complete due to repo/task parse errors.

### New reset helper
- `track_reset_file_match()`
  - Clears all four globals above.
  - Mirrors the existing `track_reset_task_parse()` / `track_reset_task_resolution()` pattern.

### New matcher
- `track_match_files_to_task()`
  - Signature: variadic changed file paths, e.g. `track_match_files_to_task "$@"`
  - Input source: positional args only; no stdin parsing and no JSON/text output.
  - Task source: `$(track_repo_root)/.track/tasks/*.md`
  - Matching basis: existing `track_globs_overlap_serialized`
  - Result contract:
    - exactly 1 open-task match → `TRACK_MATCH_CONFIDENCE='deterministic'`, matched ID populated, return `0`
    - 2+ open-task matches → `TRACK_MATCH_CONFIDENCE='ambiguous'`, all matched IDs populated, return `0`
    - no open-task matches or no changed paths → `TRACK_MATCH_CONFIDENCE='unmatched'`, matched IDs empty, return `1`
    - repo/task parsing failure → `TRACK_MATCH_ERROR` populated, return `2`

## Implementation Details
1. Add `track_reset_file_match()` near the other reset helpers.
2. Implement `track_match_files_to_task()` after `track_globs_overlap_serialized()` so it sits beside related glob logic.
3. Inside `track_match_files_to_task()`:
   - Call `track_reset_file_match`.
   - Serialize changed paths with `track_serialize_items "$@"`.
   - If the serialized changed-path list is empty, set `unmatched` and return `1`.
   - Resolve `task_dir="$(track_repo_root)/.track/tasks"`.
   - If `task_dir` does not exist, set `TRACK_MATCH_ERROR` and return `2`.
4. Iterate task files deterministically:
   - Use `find "$task_dir" -maxdepth 1 -type f -name '*.md' | sort`
   - This guarantees stable ambiguous-match ordering across runs.
5. For each task file:
   - Parse with `track_parse_task_file`.
   - If parsing fails, stop immediately, set `TRACK_MATCH_ERROR` to include the file path plus `TRACK_parse_error`, and return `2`.
   - Skip tasks whose `TRACK_status` is terminal via open-status filtering.
   - Skip tasks with empty `TRACK_files`.
   - Serialize task globs with `track_serialize_items "${TRACK_files[@]}"`.
   - Call `track_globs_overlap_serialized "$changed_files_serialized" "$task_files_serialized"`.
   - On overlap, append `TRACK_id` to `TRACK_MATCHED_TASK_IDS`.
6. After iteration:
   - Build `TRACK_MATCHED_TASK_IDS_SERIALIZED` from the collected array.
   - Set confidence and exit code from match count.

## Assumptions and Defaults
- Match scope is limited to open tasks: `todo`, `active`, `review`.
- Result transport is globals + exit code, not stdout text or JSON.
- Ambiguous matches are ordered by sorted task filename traversal.
- The matcher intentionally uses Track’s existing prefix-overlap heuristic rather than full shell glob semantics.
