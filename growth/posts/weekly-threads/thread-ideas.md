# Weekly X Thread Ideas

8 thread concepts for weeks 2-8. Each thread should be independently valuable — not just Track promotion. Goal: establish authority on "agent coordination" and "post-PM workflows."

Pick 1-2 per week. Draft in this directory as individual files (e.g., `week2-agent-chaos.md`).

---

## 1. "Agent Chaos" (Problem → Solution)
**Hook:** I watched 3 AI agents try to edit the same file. Here's what happens without coordination.
**Angle:** Show the actual chaos (merge conflicts, overwritten work, duplicated effort), then introduce Track as the fix.
**Assets needed:** Screen recording of uncoordinated agents failing.

## 2. "Why Linear Built an Agent" (Industry Analysis)
**Hook:** The real reason Linear built an AI agent (and why it won't save them).
**Angle:** Linear's agent admission = PM tools are admitting they're the bottleneck. But their answer is still "use our tool." The paradigm needs to die, not evolve.
**Assets needed:** Linear Next screenshots, comparison data.

## 3. "Folder Life" (Mass Appeal)
**Hook:** I manage my entire life with folders and markdown. Here's my system.
**Angle:** The "everything is project management" thesis. Show non-code examples: travel planning, book writing, home renovation. Make it relatable beyond developers.
**Assets needed:** Screenshots of folder structures for non-code projects.

## 4. "5 Agents No Rules" (Chaos Demo Video)
**Hook:** What happens when you give 5 agents the same codebase with no rules.
**Angle:** Record a video of 5 Conductor agents all trying to work on the same repo without Track. Then show the same scenario with Track. Before/after.
**Assets needed:** 2-3 minute video, split screen.

## 5. "PM Tools Are Just Markdown" (Provocative Thesis)
**Hook:** The project management tools you pay for are just markdown files with a UI.
**Angle:** Reverse-engineer what Jira/Linear/Notion actually store. It's structured text. You're paying $16/seat for a rendering layer over what git does for free.
**Assets needed:** Side-by-side of a Linear ticket vs a .track/ task file.

## 6. "Ported to Cursor" (Build in Public)
**Hook:** I ported Track to Cursor in 2 hours. Here's why cross-platform agent tools matter.
**Angle:** Walk through the porting process. Show that Track's markdown+bash architecture makes it trivially portable. This is why protocol > product.
**Assets needed:** Screenshots of Track working in Cursor.

## 7. "The Price of PM" (Direct Comparison)
**Hook:** Linear costs $192/year/seat. Track costs $0. Here's what you lose (nothing).
**Angle:** Genuinely fair comparison. Acknowledge Linear's strengths. Then show the math: for a 10-person team, that's $1,920/year for what a folder does.
**Assets needed:** Cost comparison graphic.

## 8. "Wedding Track" (Crossover Appeal)
**Hook:** A non-coder friend manages her wedding with `.track/`. This is the future.
**Angle:** Set up Track for a real non-code project. Show the folder structure, tasks, how an AI agent helps manage it. This is the "git for everything" thesis made real.
**Assets needed:** Screenshots of a wedding planning .track/ folder.
